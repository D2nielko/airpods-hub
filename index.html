How this works

Apple doesn’t offer a public AirPods API, so this page uses two real techniques plus a fallback:

  • BLE advertisement scan — AirPods continuously broadcast Apple’s proximity pairing message (manufacturer ID 0x004C, type 0x07), which carries left/right/case battery and charging flags in plain bytes. This needs Chrome’s chrome://flags/#enable-experimental-web-platform-features flag for requestLEScan.
  • GATT Battery Service — the standard Bluetooth battery service. AirPods don’t expose it to non-Apple hosts, but many other earbuds do, so it’s here for completeness.
  • Demo mode — simulated data so you can explore the UI anywhere.

The case only broadcasts while its lid is open or it’s actively connected — if the case tile stays empty, pop the lid open near your computer.
